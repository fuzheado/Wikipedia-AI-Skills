# Wikipedia Perennial Sources Reference

> **⚠️ This document summarizes community consensus from [WP:RSP](https://en.wikipedia.org/wiki/Wikipedia:Reliable_sources/Perennial_sources)
> as of June 2026. Consensus can change — always check the live page for the definitive current status
> of any source, especially for recently discussed or edge-case sources.**

This reference lists sources that have been *repeatedly* discussed on the
[Reliable Sources Noticeboard](https://en.wikipedia.org/wiki/Wikipedia:Reliable_sources/Noticeboard) (RSN).
A source's absence from this list does not imply reliability or unreliability — it
only means it hasn't been the subject of repeated community discussion.

---

## The Classification Tiers

| Icon | Status | Meaning |
|------|--------|---------|
| 🟢 Generally reliable | `GREL` | Reliable in most cases within its areas of expertise. Has a reputation for fact-checking and accuracy. Excluding the source requires strong arguments. |
| 🟡 No consensus / Marginal | `NC` / `MREL` | Neither generally reliable nor generally unreliable. Usable depending on context. Evaluate each use case-by-case. |
| 🔴 Generally unreliable | `GUNREL` | Questionable in most cases. May lack editorial oversight, have poor fact-checking, or be self-published. Should normally not be used; never for BLPs. |
| ⛔ Deprecated | `DEPREC` | Community consensus prohibits use as a reference for facts. May still be usable as a primary source for attributed opinions about itself (WP:ABOUTSELF). |
| 🚫 Blacklisted | `BLACK` | On the spam blacklist — links are technically blocked. Must be whitelisted before they can be used. |

### Special Topics

#### Satirical Sources

Satirical sources (e.g., *The Onion*, *The Babylon Bee*) should **not** be used as sources for facts.
They may be usable as primary sources for articles *about* the publication itself.

#### Large Language Models / AI

Per [WP:RSML](https://en.wikipedia.org/wiki/Wikipedia:RSML), content produced by LLMs (ChatGPT,
Gemini, Claude, etc.) is generally unreliable. Do not use AI-generated text as a source.

#### Paid News (India)

Paid news and undisclosed advertorials are pervasive in many Indian commercial news
organizations. Exercise caution with sources like *The Times of India* (supplements),
*Daily Pioneer* (Special section), *Hindustan Times* (Brand Post), *India Today*
(Impact feature), and *Firstpost* (Press Release News). Look for individual bylines,
check for disclosure labels, and cross-reference with independent sources.

#### Nigerian "Brown Envelope" Journalism

Undisclosed promotional articles are common across Nigerian media. Assess Nigerian
sources with extra caution for BLP coverage and notability claims.

#### Student Media

Reputable student newspapers (e.g., *The Harvard Crimson*) are generally reliable for
news on their school and local community but do **not** contribute to notability for
topics related to their home institution.

#### Self-Published Peerage Websites

The following are deprecated:
almanach.be, almanachdegotha.org, angelfire.com/realm/gotha, chivalricorders.org,
cracroftspeerage.co.uk, englishmonarchs.co.uk, jacobite.ca, royalark.net,
thepeerage.com, worldstatesmen.org

| Source is primary, secondary preferred | `{{primary source inline}}` | After the `<ref>` |

## Checking the Live List

The definitive, complete list is always at:

**https://en.wikipedia.org/wiki/Wikipedia:Reliable_sources/Perennial_sources**

To check a source not in this reference:
1. Search the [RSN archives](https://en.wikipedia.org/wiki/Wikipedia:Reliable_sources/Noticeboard)
2. If no consensus exists, the source's reliability must be evaluated on its own merits
3. Sources with clear editorial oversight, fact-checking, and correction policies are generally reliable
4. Sources that are self-published, user-generated, or lack editorial oversight are generally unreliable

## Citation Tools & Services

These tools help you find, generate, verify, and fix citations:

### Finding Sources

| Tool | What It Does | Access |
|------|-------------|--------|
| **[The Wikipedia Library](https://wikipedialibrary.wmflabs.org/)** | Free access to 80,000+ paywalled periodicals (JSTOR, ProQuest, Newspapers.com, etc.) for active editors | Requires 500+ edits, 6+ months, 10+ edits in last month |
| **[WP:Resource Exchange](https://en.wikipedia.org/wiki/Wikipedia:WikiProject_Resource_Exchange/Resource_Request)** | Request a specific source from fellow Wikipedians who may have access | Free for any editor |
| **[OAbot](https://oabot.toolforge.org/)** | Automatically finds open-access versions of paywalled articles | Toolforge (can be run on any Wikipedia page) |
| **[Google Scholar](https://scholar.google.com/)** | Academic papers and citations | Free |
| **[Internet Archive Scholar](https://scholar.archive.org/)** | Full-text search of 25+ million open-access research papers | Free |
| **[DOI Foundation](https://www.doi.org/)** | Resolve DOIs to find articles | Free |

### Generating Citations

| Tool | What It Does | Best For |
|------|-------------|----------|
| **Citoid API** (built into VisualEditor) | Auto-generates citation from URL, DOI, ISBN, PMID | One-off citations from major sites |
| **[Citation bot](https://en.wikipedia.org/wiki/User:Citation_bot)** | Expands partial citations, adds DOIs, identifiers, archive links | Fleshing out existing citations; batch processing |
| **[reFill](https://refill.toolforge.org/)** | Expands bare URLs on a page into full citation templates | Cleaning up many bare URLs at once |
| **[ProveIt](https://en.wikipedia.org/wiki/Wikipedia:ProveIt)** | User script with visual citation editor | Interactive citation building |
| **[Web2Cit](https://www.mediawiki.org/wiki/Web2Cit)** | Community-driven citation mapping for sites Zotero misses | Sites without existing translators |
| **`scripts/citoid-expand.sh`** (this skill) | CLI tool using Citoid API | Automation/scripting |
| **`assets/citoid_fetcher.py`** (this skill) | Python library using Citoid API | Programmatic batch processing |

> **⚠️ Always verify tool-generated citations.** They can misattribute authors,
> omit page numbers, use wrong template types, or include dead URLs.
> See [WP:AGCR](https://en.wikipedia.org/wiki/Wikipedia:AGCR) for common failure modes.
